| Learning | It is important to be clear about what we mean by “learning” in machine learning. Machines do not learn in the same way humans or animals do. When we first play a video game, we learn the buttons, infer the hidden rules, build expectations about what will happen next, and can usually adapt if the situation changes in an unusual way. For example, if the television were tilted 90 degrees, we would still understand that the same game is being displayed, just rotated. A machine learning model does not automatically have that kind of flexible understanding. Unless the model has been trained on rotated examples, or designed in a way that handles rotation, it may treat the tilted image as something completely different. So when we say a machine is “learning,” we do not mean that it understands the world like we do. We mean that data is being used to shape a mathematical *function* (I will explain this) so that it becomes better at producing useful outputs.|   
| Function | You are likely familiar with functions from your python work. When we talk about functions from here on out we need to be clear if we are talking about code functions or mathematical functions. Mathematical functions sound scary and look scary at first `f(x)= x + 2` -- `f(3) = 5`. What does this none sense mean? A mathematical function is a rule that takes an input and gives one specific output. ``` def my_function(x): return x + 2 ``` is analogous to `f(x) = x + 2` |
